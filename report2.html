Report 2: Malware Static Analysis (Ukraine 2022)

Yll Morina

02/05/2026

This report shows the steps I used to analyze a malware sample linked to attacks on Ukrainian organizations in 2022. I focused on static analysis and used public research to confirm what the malware is.

1. Sample Information

I started by opening the file in PE-bear to collect the hashes and confirm the file identity.

SHA-256: a64c3e0522fad787b95bfb6a30c3aed1b5786e69e88e023c062ec7e5cebf4d3e

2. VirusTotal Check

Next, I searched the SHA-256 hash in VirusTotal to confirm if it is already known and how many engines detect it.

VirusTotal Link: View Detection Page

3. File Type and Basic Details

I used Detect It Easy (DIE). It showed the sample is a PE32 file (32-bit, x86) and looks like it was built to be compatible with Windows XP. This usually means it can also run on newer Windows versions.

4. Sections and Packing Signs

I checked the PE sections and looked at entropy. Some sections had high entropy, which usually means the file is packed or encrypted. This is a common malware trick to hide the real code.

5. Imports Look Obfuscated

Imports did not look normal. They seemed missing or malformed (spoofed imports). That suggests the malware probably loads Windows APIs at runtime instead of listing them normally. This makes analysis harder.

6. I Tried FLOSS (Strings) But It Failed

I tried running FLOSS on the sample in normal mode and also with verbose options. Every time it crashed with a UnicodeDecodeError. That’s another sign the file is packed or intentionally malformed.

7. PEStudio Strings (Most Were Gibberish)

I opened the file in PEStudio and reviewed the strings. Most were unreadable, but near the end there were interesting readable strings related to disk paths and system locations. This can point to disk-level activity.

8. Certificate / Signature Info

PEStudio showed a digital certificate, but it did not verify. Malware sometimes uses stolen or abused certificates to look legitimate.

9. Correlating With Public Research (CISA)

At this point I was still not fully sure what the malware was doing, so I researched the hash and found a CISA advisory that explains the malware campaign used against Ukraine in 2022.

CISA Advisory: AA22-057A

10. Final Summary

Based on the packed/obfuscated PE structure, broken imports, unreadable strings, and the matching public reporting from CISA, I concluded this sample is part of the destructive malware activity described in the advisory.
